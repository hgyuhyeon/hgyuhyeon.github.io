Overview
A website that recommends musics based on the image using AI.
AI를 이용한 이미지 기반 음악 추천 웹 서비스
Tech stack
Python, Django
  • Selected the Django framework in Python to integrate the AI model into the system.
  • AI 모델을 시스템에 내장시키기 위해 Python 프레임워크인 Django를 선택함.
MySQL
  • Chose MySQL over the default SQLite in Django.
  • MySQL made database maintenance easier, and writing SQL queries directly was a better fit for the development environment at the time
  • Django의 내장 SQLite 대신 사용.
  • 수월한 데이터베이스 유지보수와, sql문을 직접 작성해서 데이터를 불러오는 것이 당시 개발환경에 훨씬 편안한 상태였음.
Nginx, Gunicorn, AWS
  • Chose these technologies due to their excellent compatibility with Django and Python projects, ensuring smooth deployment and reliable performance.
  • Django 및 Python 프로젝트와의 뛰어난 호환성 때문에 선택하였으며, 안정적인 배포와 성능을 보장 가능.
Docker, Git
  • Applied these tools for collaboration; Docker reduced setup time by over 70%.
  • Used Git to separate repositories for front-end, back-end, and AI, merging them later for the final build.
  • 협업을 위해 적용한 툴로, Docker 사용으로 셋업 시간을 70% 이상 절약.
  • Git으로 프론트, 백엔드, AI repository를 나누어 작업한 뒤 추후 병합.
Contributions
  • Architected the back-end system, designing the core APIs that handled user data, image uploads, and music recommendations using Django.
  • Improved API performance by optimizing Gunicorn worker settings, reducing response times from 10 seconds to 2 seconds, resulting in an 80% efficiency boost.
  • Deployed the entire application to AWS EC2 using Docker Compose, integrating all system components (back-end, front-end, AI, database) into a unified and scalable solution.
  • Encountered a MySQL execution error during Docker Compose testing.
    • Cause: The MySQL container was starting slower than other containers.
    • Solution: Implemented healthcheck and depends_on in Docker to ensure proper startup sequencing and resolve the issue.
  • 백엔드 시스템을 설계하고, 사용자 데이터 및 사진 업로드, 음악 추천을 처리하는 핵심 API 설계 및 구현.
  • Gunicorn 설정 최적화를 통해 API 성능을 10초에서 2초로 단축하여 80% 효율성 향상.
  • Docker-compose를 사용하여 애플리케이션을 AWS EC2에 배포, 백엔드, 프론트엔드, AI, 데이터베이스를 통합하여 확장 가능한 솔루션 구축.
  • Docker-compose 테스팅 중 MySQL 서버가 정상 실행되지 않는 현상 발생
    • 원인은 MySQL 서버의 런칭 속도가 Django보다 느려서 Django의 마이그레이션이 정상 적용되지 않았던 것.
    • Docker의 depends_on, healthcheck를 활용하여 해결.
